# peakflow-releases

## Purpose
- This repo is for customer-ready releases.
- Keep wording plain.
- Do not add inflated product copy.

## Rules
- Only ship-ready code belongs here.
- No unfinished features.
- No scratch files.
- No local tooling folders.
- No secrets.

## Workflow
- Build and test in the private dev repo first.
- Move code here only when you intend to release it.
- Treat pushes here as release actions, not backup actions.
